Glossary¶

Blob object

Represents the contents of a file. Reference Blob object.

Commit object

Represents a snapshot of the state and point in time in the repo history. Contains a Tree object. Reference Commit object.

Git Directory

This is the .git folder in your Git Repository. Reference Git Directory.

Git Database

The Git Object storage area .git/objects/ is a key-value store containing all of the Git objects.

Git Index

The staging area where you can prepare files before they are committed into the Git Database. Reference Git Index.

Git Repository

Reference: ref:core-concepts:Git Repository.

Tag object

Represents a reference to a particular Git Object. Often used to tag a Commit object. Reference Tag object.

Tree object

Represents the files in a directory. Can contain Blob objects and Tree objects. Reference Tree object.

Worktree

The working directory where Git files are checked out to. This is where you modify and update files before staging the files into your Git Index. Reference Worktree.
